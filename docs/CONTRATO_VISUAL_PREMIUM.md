# Contrato Visual Premium

Regra central: uma imagem oficial de patch so entra como imagem oficial quando o jogo entrega, no minimo, os sistemas e a sensacao principal que ela promete.

## Classificacao de imagem

1. **Imagem oficial de patch**
   - Precisa representar gameplay real ou sistema ja implementado.
   - Pode ser mais cinematografica que a camera do jogo, mas nao pode inventar mecanica inexistente.
   - Deve ter lista de equivalencia em codigo/docs.

2. **Previa visual**
   - Pode mostrar futuro desejado.
   - Precisa ser marcada como conceito/preview.
   - Nao deve aparecer como promessa de update entregue.

3. **Referencia interna**
   - Serve para direcao de arte.
   - Nao aparece para jogador ate virar sistema.

## Checklist antes de publicar imagem oficial

- Cenario aparece ou tem bloco jogavel equivalente.
- Objetos importantes existem no mapa/interior.
- NPC, monstro, pet, montaria ou item mostrado tem funcao, dialogo, loot, quest ou interacao.
- Luz, horario e atmosfera existem no jogo.
- Se a imagem mostra economia/casa/viagem/hunt, o sistema correspondente esta implementado.
- Se algo ainda nao existe, a imagem deve ser rebaixada para previa visual.
